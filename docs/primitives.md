# Core Primitives

GSD-AI starts with a small set of primitives.

## Goal

A durable outcome the user is trying to move forward.

## Project

The primary container for a body of work connected to a goal, initiative, deliverable, or responsibility.

## Context contract

The compact, AI-readable state of a project:

- purpose
- current status
- active goals
- decisions
- risks
- dependencies
- actions
- open questions
- key references
- scope/privacy notes

## Signal

A structured piece of work state extracted from messy context.

Default signal types:

- context shift
- risk
- dependency
- decision
- open decision
- mitigation
- action
- escalation

## Action

A signal or task that requires execution.

## Time block

A calendar commitment attached to an action/task.

## Scope

A boundary for where information may be used:

- private
- team
- client
- public-safe
- sensitive-never-share

## Evidence

Proof that an AI-generated claim or write is grounded and correct.

Evidence chain:

```text
Intent → Claim → Witness → Verdict → Commit → Audit
```
